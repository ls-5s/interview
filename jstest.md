## 如果要做优化，CSS提高性能的方法有哪些？
**为什么要优化**
每一个网页都离不开css,但是很多人又认为，css 是主要完成页面的布局，
像一些细节的优化，就不怎么考虑，这个是错误的。
作为页面渲染和内容展现的重要环节，css影响着用户对整个网站的第一体验
1. 内联关键css 样式加载
在html 解析完了，可以马上渲染css 样式(首页的基础样式，导航，首页内容布局)

外联是遇到css 才下载，渲染，回导致时间浪费（页脚，或者下拉菜单）
减少渲染时间和避免html 文件过大(加载慢，解析时间长)
2. 使用link 不使用import 
因为imort 它必须等待第一个css 下载解析后，才可以执行第二个，这样会造成页面的
卡顿
3. 减少使用昂贵的属性
如果使用太多，box-shadow,在回流，可能会降低浏览器的性能
4. 资源的压缩
去重掉css CSS 中冗余内容（注释、空格、换行），合并重复的规则，就是减少字节传输，加快css 下载下载速度
资源压缩的核心是剔除 CSS 中冗余内容（注释、空格、换行），合并重复规则，甚至简化属性值（如缩写margin: 10px 20px代替多行），让文件体积变小。
除了 webpack 等构建工具（通常通过css-minimizer-webpack-plugin等插件实现），也可以用专门的 CSS 压缩工具（如 csso、cssnano），压缩效率更针对性。
压缩后直接减少传输字节数，尤其在弱网环境下，能显著加快 CSS 下载速度，间接缩短渲染阻塞时间，同时节省用户带宽。

## 面试官问:那你在说说当后端传给你很多数据的时候，你该怎么处理？

1. 在请求阶段
分页加载，就是每一页限定有多少条数据
在点击下一页的， 在从后端那数据
2.滚动加载：
监听页面滚动事件（或容器滚动），当用户滚动到接近底部时，自动请求下一批数据并追加到列表中（类似朋友圈、微博的加载逻辑）。这里要做节流处理，
3. 安字段放回
   
优化数据解析和存储
- 数据缓存
对于已经加载的数据，可以做一缓存
内存缓存：用Map或对象存储已加载的分页数据（如cache = { page1: [...], page2: [...] }），切换页码时直接从缓存读取；

- 数据筛选与过滤
前端用户只保留当前需要展示/交付的数据，去掉无用的信息，
用户搜索时，在前端对已加载数据进行本地筛选（避免频繁请求后端）；
对嵌套过深的对象进行 “扁平化” 处理（如{ user: { name: 'xxx' } }转为{ userName: 'xxx' }），减少渲染时的属性查找成本。
- Offscreen 数据计算（Web Workers）
若需要对大量数据进行复杂计算（如排序、统计、格式化），将计算逻辑放入Web Worker中执行，避免阻塞主线程（JS 是单线程，大量计算会导致 UI 卡顿）。

## 面试官：说一下 GET 和 POST 的区别？
1. 
- GET:  “获取” 资源，用于从服务器请求已存在的资源（如查询数据、加载网页）。
- POST：“提交” 资源，用于向服务器发送数据，通常会导致服务器上的资源被创建或修改（产生副作用）

## 面试官：如何理解UDP 和 TCP? 区别? 应用场景?(没有看完，接着看)

**UDP**
一种 “无连接、不可靠” 的传输协议。它像 “发短信”—— 发送方直接把数据打包成 “数据报”，不确认接收方是否在线，也不保证数据能到达、到达顺序是否正确，发完就完事，开销极小，速度快。
