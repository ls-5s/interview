# 面试官：说说 JavaScript 中内存泄漏的几种情况？
先说说什么垃圾回收机制

原理：垃圾收集器会定期（周期性）找出那些不在继续使用的变量，然后释放其内存

======================================

. 标记清除法（Mark-and-Sweep）

：标记阶段
从 “根对象”（如全局对象 window）出发，遍历所有可达的对象（即能被直接或间接引用的对象），并为这些对象打上 “存活” 标记。
例如：全局变量 a 引用对象 obj，obj 的属性 b 引用对象 obj2，则 a、obj、obj2 都会被标记为 “存活”。


步骤 2：清除阶段
遍历内存中所有对象，清除所有未被标记（即 “不可达”）的对象，释放其占用的内存。
同时，引擎会整理内存碎片（将存活对象的内存地址连续排列，减少内存碎片化）。
