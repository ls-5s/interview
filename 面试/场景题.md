# 你在这个项目中做了哪些前端性能优化？分别带来了什么效果？

1. 骨架屏（Skeleton）优化加载体验
做法：针对简历解析、LLM 评分这类耗时操作，用骨架屏替代传统的 loading 转圈 / 空白页，页面结构占位展示；
效果：彻底消除用户等待焦虑，视觉上感觉页面加载更快，无空白卡顿，用户体验提升非常明显。
2. 异步非阻塞 + 轻量轮询，解决接口超时
做法：简历上传后，接口不等待解析和评分，直接返回成功；后台异步处理耗时任务，前端每隔 1 秒轮询查询任务状态；
效果：接口响应速度从 5~8 秒 降至 0.5 秒内，用户无需等待，页面全程不阻塞、不卡死。
3. 接口请求防抖 + 防重复提交
做法：
① 简历搜索框添加500ms 防抖，避免输入时频繁请求接口；
② 封装请求函数，拦截重复点击、重复上传操作；
效果：减少60% 以上的无效接口请求，降低服务器压力，搜索 / 操作响应更流畅。
4. 组件 / 路由懒加载（按需加载）
做法：UI 组件库按需引入（不全局加载）、简历详情 / 评分面板等页面路由懒加载；
效果：项目首屏打包体积减小 40%，首屏加载速度大幅提升，打开页面更快。
5. 图片 / 静态资源懒加载
做法：页面所有图标、图片默认懒加载，只有进入可视区域才加载；
效果：减少首屏资源请求数，降低带宽消耗，移动端 / 弱网环境下加载速度提升明显
6. 简历列表渲染优化
做法：简历列表只渲染可视区域的数据，避免上百份简历同时渲染造成 DOM 过载；
效果：长列表滚动无卡顿，页面渲染性能提升，即使批量展示简历也流畅运行。
