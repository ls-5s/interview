# 进程间的通信

进程间通信（IPC）是指不同进程之间交换数据和进行协作的机制，常见方式包括管道、消息队列、共享内存、信号量、Socket 等，核心是在隔离性与通信效率之间做权衡。
为什么会有ipc ?
进程是资源隔离的
每一个进程有独立的地址空间
所以在天然无法直接访问彼此的数据
常用的ipc 方式有

1. 管道
他是 半双工
只能用于有亲缘关系的进程（匿名管道）

优点：

简单
内核支持

缺点：

不能跨机器
数据量有限
2. 消息队列
特点：

以“消息”为单位传输
内核维护队列
优点：

解耦
支持异步通信

缺点：

有拷贝开销
容量有限
3. 共享内存（Shared Memory）

特点：

多个进程共享同一块内存

优点：

🚀 最快 IPC（零拷贝）
高性能

缺点：

需要自己控制同步（容易出错）

6️⃣ Socket（重点🔥🔥🔥）

特点：

支持跨进程 + 跨机器
基于网络协议（TCP / UDP）

优点：

通用性最强
分布式系统核心

缺点：

性能低于共享内存

# 进程和线程的区别

进程是资源分配的基本单位，线程是 CPU 调度的基本单位，本质区别在于是否共享地址空间。
进程
一个正在运行的程序
拥有独立的地址空间，内存，文件描述等
线程

进程中执行的单元
多个线程共享进程资源，但有各自的执行栈

两者的差别
进程拥有独立的堆栈空间和数据段，需要分配独立的地址空间，开销大
线程开销小，切换快，没进程安全，从通信机制上看，线程可以共享数据段

【4️⃣ 举例（一定要说🔥）】
浏览器：
一个 Tab 通常是一个进程（隔离性强）
Node.js：
用 PM2 开多个 instance，本质是多进程
Web Worker：
独立执行环境（通常基于线程），通过消息通信，不直接操作 DOM
