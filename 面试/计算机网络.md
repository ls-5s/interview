# HTTP 几个版本的区别

- HTTP/0.9 - 单行协议
只有 GET 请求行，无请求头和请求体
只能传输 HTML 文件，以 ASCII 字符流返回
无响应头
- HTTP/1.0 - 多类型支持
支持多种文件类型传输，不限于 ASCII 编码
引入请求头和响应头( key-value 形式)
每个请求都需要建立新的 TCP 连接
- HTTP/1.1 - 持久连接
引入持久连接( keep-alive )：一个 TCP 连接可传输多个 HTTP 请求
默认开启 keep-alive，通常限制 6-8 个并发连接
存在队头阻塞问题：前面的请求阻塞会影响后续请求
引入 Host 字段，支持虚拟主机
引入 Chunk transfer 机制处理动态内容长度
- HTTP/2 - 多路复用
一个域名只使用一个 TCP 长连接
引入二进制分帧层，实现多路复用
可对请求设置优先级
引入 HTTPS(HTTP + TLS) 加密

# HTTP 常见的状态码

200 请求成功，请求在服务端被正确处理
204 响应成功，没有数据
205 服务器处理成功，浏览器应重置文档视图
206 服务器成功处理了部分get请求
301 资源永久重定向
302 资源临时重定向
303 让你查看其他地址
304 请求的资源没有修改，服务端不会返回任何资源，协商缓存
400 请求语法错误，服务器看不懂
401 请求没有携带信息，比如 token 认证失败
403 请求被拒绝、敏感词
404 找不到资源
500 服务器内部错误，无法完成请求
501 服务器不支持当前请求所需的功能
503 服务器系统维护或者超载，暂时无法处理客户端的请求

# GET 和 POST 请求的区别

GET：查数据、URL 传参、可见、缓存、幂等、大小有限
POST：改数据、body 传参、安全、不缓存、不幂等、大小无限制

# 什么是跨域？如何实现跨域通讯？

跨域 = 浏览器的同源策略限制不同源之间的资源访问；跨域通讯 = 绕过或利用规则实现不同源的数据交互。
📌 同源策略（Same-Origin Policy）

浏览器的安全机制，要求三者必须一致：

协议（http / https）

域名（example.com）

端口（80 / 3000）
✅ 【2. 为什么会有跨域（Why）】

本质是为了防止：

XSS 攻击

CSRF 攻击

恶意网站读取用户敏感信息（cookie / token）

👉 浏览器主动限制，而不是服务器限制

🚀 1️⃣ CORS（最主流 ⭐⭐⭐⭐⭐）
📌 原理

服务器加响应头，告诉浏览器“我允许你跨域”

Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST
Access-Control-Allow-Headers: Content-Type
📌 分两种请求
✔ 简单请求

直接发

✔ 复杂请求（会预检 OPTIONS）

浏览器先问服务器：

OPTIONS /api

👉 服务器同意才发正式请求

🚀 2️⃣ JSONP（古早但面试必问 ⭐⭐⭐）
📌 原理

利用 <script> 标签不受跨域限制

<script src="http://api.com/data?callback=fn"></script>

服务器返回：

fn({ data: 123 })
❗ 缺点

只能 GET

不安全（容易 XSS）

🚀 3️⃣ 代理（开发最常用 ⭐⭐⭐⭐⭐）
📌 原理

浏览器 → 自己服务器 → 目标服务器

👉 浏览器不跨域，但服务器之间可以

📌 常见方式
Vite / Webpack
server: {
  proxy: {
    '/api': {
      target: '<http://backend.com>',
      changeOrigin: true
    }
  }
}

👉 面试加分点：
生产环境用 Nginx 反向代理

🚀 5️⃣ WebSocket（天然跨域 ⭐⭐⭐⭐）
📌 原理

WebSocket 不受同源策略限制

const ws = new WebSocket("ws://example.com")

👉 适用于：

实时聊天

推送系统
